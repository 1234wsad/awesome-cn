## Web

-[Firebase UI] [firebase-ui]-FirebaseUI是用于Web的开源JavaScript库，它在Firebase SDK之上提供了简单，可自定义的UI绑定，以消除样板代码并推广最佳实践.
-[React的Firebase UI] [fb-ui-react]-FirebaseUI Web的React Wrapper.
-[GeoFire for JavaScript] [geofire]-使用Firebase进行实时位置查询.
- 💡 [FirePad][firepad] - Collaborative Text Editor Powered by Firebase.
-[Ember Fire] [ember-fire]-适用于Firebase的官方Ember数据适配器.
-[Firebase Dart] [fb-dart]-Firebase的Dart包装器.
- 🔌 [PolymerFire][polymer-fire] - Polymer Web Components for Firebase.
-[VueFire] [vue-fire]-Vue.js的Firebase绑定.
-[Angular Fire 2] [ng-fire]-Firebase和Angular的官方库.
-[Re-base] [rebase]-中继启发式库，用于构建React.js + Firebase应用程序.
 -[React Redux Firebase] [react-redux-fb]-Firebase的Redux绑定.  包括与React一起使用的高阶组件.
-[GatsbyJS Firebase数据源] [gatsby-fire]-使用Gatsby将Firebase数据直接查询到静态生成的页面中.
 -[Apollo Link Firebase] [apollo-link-fb]-为RealtimeDB提供本地GraphQL接口.  数据库本地同步到设备，Apollo Link提供对本地数据库的查询.
-[针对Firebase的BuckleScript绑定] [bs-fb]-适用于ReasonML项目的Firebase的BuckleScript绑定.
 -[Angular Firebase PWA] [ng-firestarter]-是由Firebase驱动的Angular PWA.  它可以作为学习此堆栈并推出更复杂功能的基础.
 -[FireSQL] [firesql]-使用SQL语法查询Firestore.  发出获取所需数据所需的最少查询量.
-[托管版本历史记录] [hosting-version-hist]-自动删除旧版本的网站部署.
-[Firestorter] [firestorter]-使用MobX（也用于react-native），以零的努力在React中使用Firestore.
-[Nextbase] [nextbase]-适用于想要快速启动项目的开发人员的Next.js，Redux和Firebase样板.
-[React Firebase Hooks] [react-firebase-hooks]-用于Firebase服务的React Hooks.

## CLI & Editor

-[Firebase工具] [fb-tools]-Firebase命令行工具.
-[Firebase CI] [fb-ci]-用于持续集成的简化Firebase交互.
-[VSFire] [vsfire]-VSCode扩展，用于使用Firestore安全规则和索引突出显示语法和完成代码.
-[Firebase Firestore代码段] [fb-firestore-snippets]-包含VS Code编辑器中Firebase和Firestore的代码段.
-[Fuego] [fuego]-Firestore客户端CLI支持带有过滤和分页功能的文档添加/更新/查询.
-[Firestore Rules Generator] [firestore-rules-gen]-基于Google协议缓冲区格式的Cloud Firestore官方（但实验性）Firebase规则生成器.
-[Firepit] [firepit]-Firepit是Firebase CLI的一个独立的可移植版本，没有任何依赖（包括Node.js）.
-[Fireward] [fireward]-易于使用的Firestore规则语言，类似于Firebase Bolt.
